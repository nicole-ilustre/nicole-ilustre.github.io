    • Command Line Primer

    In a couple of sentences, how would you describe the command line in plain English? Can you think of an analogy for it? Command line is another form of communication or another way of navigating and controlling your computer. Most people are used to navigating through the visual navigation where you click on arrows and buttons. Command line does the exact same things, the only difference is that you are communicating with your computer through plain text. The only analogy I can think of is following a recipe when cooking. GUI is like watching a video of the dish being cooked and seeing visually the steps taken. Whereas, command line is like following a recipe from a book that is text-based and no video or photo. All instructions are in text.

    Install and Explore Git

    Whats the difference between git and GitHub? Git is the local software you use to control and navigate around your own local repository. This is mostly used for individual projects. Github is the cloud-based platform used to store and share work and used for collaborations with other programmers as it has features useful when sharing work. Commiting is like saving a checkpoint in a game which is a save point you can come back at any time.
    Can you think of an analogy to describe them? It is the same as storing photos in your devices and storing photos on the cloud such as Google photos. You can create an album online and invite people to add photos in your shared album but they cannot do that in your device album.     Branch, Pull, Merge

    What is main? The master branch of a project where it is at its best condition.
    Why create a Branch? To be able to work on projects and make changes without messing up the master branch
    How would you describe stage and commit to your non-tech-savvy friend? Commit is saving a point in your work where you can revert back to. You'd want to commit for lots of reasons. Some of them are 1. when you finish working on a feature and you want to save your work at that stage so you can carry on working on different features and not worry about sabotaging the previous work you did. 2. If you want to try something new or different but aren't sure if it's going to work or not, you want to save your current work progress so you can revert back to it if needed. 3. You also want to keep a record of your work.
